如何將資料開放出來¶

這一段落為這份手冊的核心部分。我們將告訴您關於資料擁有者可以如何將資料開放出來的實際並且詳細的建議。我們將從最基本的部分開始，但是也會提醒您一些需要注意的地方。最後，我們將會討論一些在資料開放的程序中可能會出現的微妙問題。

以下是我們對於將資料開放出來的三個主要建議：

  • 越簡單越好 從少，簡單，與快速開始。沒有人說所有的資料都要現在，馬上，全部都一起開放。先從開放一筆資料集開始，或是一個大資料集的某一部分資料也可 – 當然如果您可以開放越多資料那就會越好。

    Remember this is about innovation. Moving as rapidly as possible is good because it means you can build momentum and learn from experience – innovation is as much about failure as success and not every dataset will be useful.

  • 盡早與頻繁地吸收有興趣的人 盡可能早點將對於資料的使用或是重複使用的族群或是可能的對象吸收進來，同時越頻繁越好。不管他們是一般公民，商業人士或是程式開發者。這將確保您接著要推出的服務能有最大的效能和影響。

    我們要時常記得，絕大部分的資料不會直接就交由最終使用者使用。這些資料一般都會先透過一個資訊中間者來傳送。這些資訊中間商的工作就是將資料取回並轉換或是與其他資料重組後再呈現出來。例如，我們絕大部分人對於 GPS 定位系統的大量資料沒有那種需求或是不想要處理。我們比較樂於使用地圖。所以，先將資訊中間者拉進來。他們將可把開放出來的資料重新使用和整理後重新推出新產品。

  • 回應一般常見的恐懼與誤解. 這一點對於那些在大型企業或是組織內工作者或是與這些組織有何做關係者尤其重要，例如政府單位。將資料開放將會帶來許多的疑問和恐懼。所以 (a) 找出最主要的問題，和 (b) 盡可能越早回應將會越好。

在資料的開放上有四個主要的步驟，我們將會在接下來的章節裡面針對每一個步驟詳細解釋。這四個步驟的順序並沒有硬性的規定 - 很多步驟和程序都可以同時進行。

  1. 選擇您的資料集 選擇一個你預期要開放釋出的資料集，同時請記得，您可以（或是有需要）重新回到這一步驟如果在接下來的步驟中遇到阻礙或是困難。

  2. 採用一條開放授權

    1. 確定該資料享有哪一種智慧財產權的保護。
    2. 選用一個適合的 ‘open’ 開放授權將這些權限釋放出去並遵守在前一個章節關於「什麼是開放資料」中提出的開放的定義
    3. 註：如果您沒辦法找到一個恰當的授權模式，請回到步驟一重新選擇另一組資料集

  3. 將資料釋放出來 - 以原始和可用的模式釋出。您或許也需要考慮以另外一種方法，例如透過 API 的使用來釋放出這些資料。

  4. 讓資料可被搜獲 - 將資料放上網路和或許建立一個目錄中心列出所有你開放出來的資料集。

  • 選擇您的資料集
  • Apply an Open License (Legal Openness)
  • 將資料釋出與可得 (技術層面上的開放性)
  • 讓資料可被尋獲。
